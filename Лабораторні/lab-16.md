# Лабораторна робота №16. Робота з графічним клієнтом GitKraken

[Перелік усіх робіт](README.md)

## Мета роботи

Навчитися працювати з графічним клієнтом GitKraken, виконувати базові операції з Git-репозиторіями, створювати коміти, гілки, виконувати злиття, працювати з віддаленими репозиторіями GitHub, створювати Pull Requests та вирішувати конфлікти у візуальному інтерфейсі.

---

## Теоретичні відомості

### 1. GitKraken

GitKraken — це кросплатформений графічний Git-клієнт, який спрощує роботу з Git завдяки інтуїтивному інтерфейсу.  
Основні переваги:

- візуалізація історії гілок;
- зручне створення комітів і гілок;
- робота з GitHub, GitLab та іншими сервісами;
- просте створення Pull Requests;
- візуальне вирішення конфліктів.

### 2. Інтерфейс GitKraken

Інтерфейс складається з:

1. Repository Management — панель управління репозиторіями.  
2. Left Panel — локальні гілки, remote, теги, файли.  
3. Commit Graph — візуальний граф комітів і гілок.  
4. Commit Panel — staging, unstaging, diff, створення комітів.  
5. PR & Issues — інтеграція з GitHub.

![Скріншот 1: головний інтерфейс GitKraken.](./img/016/016-1.png)

### 3. Підключення GitHub

GitKraken підтримує авторизацію через GitHub OAuth, що дає можливість:

- клонувати репозиторії;
- створювати нові репозиторії;
- працювати з Issues та Pull Requests.

### 4. Основні операції GitKraken

У GitKraken можна виконувати:

- створення та перемикання гілок;
- перегляд diff та історії;
- створення комітів;
- push/pull;
- створення PR;
- merge та вирішення конфліктів.

## Завдання

1. Встановити GitKraken та виконати авторизацію через GitHub.  
2. Клонувати власний репозиторій.  
3. Створити окрему гілку.  
4. Внести зміни у README.md та створити коміт.  
5. Виконати push гілки на GitHub.  
6. Створити Pull Request у GitKraken.  
7. Виконати merge PR.  
8. Видалити гілку після злиття.
